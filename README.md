# 📊 Data Warehouse and Analytics Project

Welcome to my **Data Warehouse and Analytics Project** repository! 🚀

This project demonstrates the design and implementation of a modern **SQL Data Warehouse** solution for analytical reporting and business intelligence. It follows industry best practices in **Data Engineering**, **Data Modeling**, and **SQL-based Analytics**.

The project is built as part of my Data Analyst portfolio to showcase practical experience with data warehousing concepts and SQL development.

---

# 🚀 Project Requirements

## 🏗️ Building the Data Warehouse (Data Engineering)

### Objective

Develop a modern SQL Data Warehouse using Microsoft SQL Server to consolidate sales data from multiple business systems into a centralized repository that supports analytical reporting and informed business decision-making.

### Specifications

- **Data Sources:** Import data from two source systems (ERP and CRM) provided as CSV files.
- **Data Quality:** Cleanse, validate, and resolve data quality issues before analysis.
- **Integration:** Combine data from multiple sources into a unified analytical data model.
- **Scope:** Focus on the latest available dataset; historical tracking is not required.
- **Documentation:** Provide clear documentation of the architecture, data model, ETL process, and project workflow.

---

## 📈 BI: Analytics & Reporting (Data Analytics)

### Objective

Develop SQL-based analytical solutions to generate meaningful business insights from the Data Warehouse.

### Key Business Insights

- Customer Behavior Analysis
- Product Performance Analysis
- Sales Trend Analysis
  
These insights support business stakeholders in making informed, data-driven decisions.

# 🏛️ Data Warehouse Architecture

The project follows the **Medallion Architecture**:

```
Source Systems
     │
     ▼
 Bronze Layer
(Raw Data)
     │
     ▼
 Silver Layer
(Cleaned & Transformed Data)
     │
     ▼
 Gold Layer
(Business-ready Data Model)
     │
     ▼
Analytics & Reporting
```
